# Hall of Automata — Architectural Context

> Pre-synthesized map for automata dispatched to this repo. Read before any file exploration.
> Maintained as a living document — update when codebase structure changes.

---

## Entry points

| File | Purpose |
|------|---------|
| `.github/workflows/invoke.yml` | Primary dispatch — triggers on label, issue_comment, pr_review, or workflow_call |
| `.github/workflows/hall-ci-loop.yml` | CI re-dispatch loop — triggers on check_suite completion |
| `.github/workflows/onboard-automaton.yml` | New automaton onboarding flow |
| `.github/workflows/onboard-invoker.yml` | New invoker onboarding flow |
| `.github/workflows/hall-cleanup.yml` | Cleanup after PR merge/close |
| `.github/workflows/weekly-reset.yml` | Weekly quota counter reset |
| `.github/workflows/retry-queued.yml` | Nightly retry of hall:queued issues (03:00 UTC) |

---

## Dispatch flow

**invoke.yml:** detect context → select invoker → authorize → inject persona → run agent → post-dispatch

**hall-ci-loop.yml:** detect agent from failing check suite → select invoker → resolve retry/escalate → inject CI context → run agent

**Persona injection:** `automaton_base.md` + `roster/<slug>.md` are concatenated into `CLAUDE.md` in the runner workspace. Never commit `CLAUDE.md`.

## Key files and directories

| Path | Contents |
|------|---------|
| `agents.yml` | Live agent catalog — source of truth for capabilities, `max_turns`, `max_retries`, team |
| `agents/automaton_base.md` | Base contract prepended to every persona at dispatch |
| `agents/automaton_template.md` | Template for new persona files |
| `roster/<slug>.md` | Persona files — character, domains, scope per agent |
| `routing.yml` | Domain-to-agent routing rules used by detect step |
| `actions/<name>/action.yml` | Reusable composite GitHub Actions |
| `scripts/*.sh / *.js` | Step implementations called by workflow steps |

## Composite actions

| Action | Role |
|--------|------|
| `actions/dispatch` | Runs the Claude agent via SDK |
| `actions/authorize` | Validates invoker team membership before dispatch |
| `actions/counter` | Reads/increments weekly quota counters per invoker |
| `actions/memory` | Save/restore task memory (GitHub cache, keyed by PR number) |
| `actions/status-card` | Updates the Hall status comment on the triggering issue |
| `actions/post-dispatch` | Audit logging after agent run completes |
| `actions/cleanup` | Post-merge branch/label cleanup |

## Agent token model

Automata do not hold OAuth tokens. Each dispatch pool-selects an invoker (`invoker/<handle>` GitHub environment) whose `CLAUDE_CODE_OAUTH_TOKEN` is used. Usage is tracked per invoker via environment variables `HALL_USAGE_COUNT` / `HALL_WEEKLY_CAP`.

---

## Hard constraints

- `CLAUDE.md` in runner workspace = automaton_base.md + persona; never commit it
- `.hall/` in runner workspace = read-only Hall repo checkout at dispatch time
- New automaton: `agents.yml` entry + `roster/<slug>.md` committed together in one push
- Agent PR branch convention: `hall/<agent>/issue-<N>`
- Dispatch result file: `.hall/dispatch-result.json` — written by agent, read by CI, never committed
